# Structure Gate

Run this after the first full draft and before prose polishing.

## Introduction check

The introduction must:

1. open with a hook (provocative claim, anecdote, contrast, statistic, or question)
2. frame the problem and define the argumentative question
3. delimit scope (what the essay covers and excludes)
4. state the thesis (contestable, last sentence of introduction)
5. close with a roadmap sentence

If the roadmap or thesis is missing, the structure gate fails.

## Essay type validation

Apply the rules matching the selected essay type. Default: argumentative.

- **Argumentative**: one analytical dimension per section; do not collapse dimensions
- **Comparative**: explicit criteria defined early; each subject evaluated against same criteria; side-by-side synthesis, not separate summaries
- **Chronological**: temporal markers bound each phase; causal analysis links phases; transitions explain change
- **Thematic**: each theme applies one analytical lens across the full topic; convergence section identifies intersections
- **Problem-Solution**: problem defines scope, severity, stakeholders; root causes distinct from symptoms; solution includes feasibility evidence

## Conclusion check

The conclusion must:

1. restate the thesis in new language (not copy-pasted from introduction)
2. synthesize how body sections converge to support the thesis
3. acknowledge at least one limitation
4. state broader implications beyond the essay's scope

If the conclusion introduces new analytical dimensions, the structure gate fails.

## Counterargument check

Verify counterargument contains: (1) steel-manned opposing position, (2) concession of valid points, (3) distinction explaining why thesis holds, (4) reaffirmation. May appear as standalone section or inline. Flag if any component is missing.

## Transition checks

Between major sections, verify:

1. what problem the current section addresses
2. what conclusion it establishes
3. why the next section must follow from it

If the bridge is implicit only, add a transition sentence using strategies from `references/transitions.md`.

## Failure patterns

Flag and rewrite when you see:

- one paragraph doing evidence + regulation + ethics at once
- introduction with thesis but no route through the essay
- conclusion introducing new analytical dimensions
- counterargument that only presents a weak straw-man version
- comparative essay treating subjects in isolation without shared criteria
- chronological essay listing events without causal links
- thematic essay where themes are sequential, not analytical

Roadmap requirement -> see SKILL.md Hard Rule #2.
